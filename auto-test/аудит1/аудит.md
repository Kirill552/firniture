Наблюдения и замечания (A11y/UX):
- Диалоги: для контента диалога (например, глобальный поиск, сравнение фурнитуры) отсутствует связанное описание (`aria-describedby`) — в консоли предупреждение «Missing Description or aria-describedby for DialogContent». Требуется добавить либо `Description`, либо привязку к скрытому описанию.
- Диалог «Настройки подключения 1С»: в консоли предупреждение «Password field is not contained in a form». Поле пароля должно находиться внутри формы с корректной семантикой (или нужно подавить предупреждение, если осознанно без формы).
- Глобальный поиск: кнопка «Открыть глобальный поиск (⌘K)» не всегда доступна для программного клика по тексту. Нужны стабильные селекторы/aria-label для автотестов и улучшения доступности.
- Таблица заказов: автоклик по чекбоксам строк не отработал (clicked-0). Проверить, не заблокированы ли элементы (overlay/pointer-events), соответствие `label` → `input`, доступные имена и таргеты по клавиатуре.
- Интеграции: кнопка «Проверить» меняет статус на «Подключение…» — ок. Диалог настроек открывается, но нет явного `aria-labelledby/aria-describedby` у содержимого; добавить заголовок/описание связью ID.
- CAM в мастере: «Сгенерировать DXF» меняет состояние на «Генерация DXF…» — ок. Нужен недоступный (disabled) прогресс-индикатор и текстовая альтернатива для screen reader.
- Общие: у декоративных иконок стоит проверить `aria-hidden="true"` и `focusable="false"`; убедиться, что фокус-трап в диалогах корректный и Escape закрывает диалог.
