## Папка автотестов фронтенда (Playwright MCP)

Это рабочая зона для e2e/UI‑аудитов и автопроверок интерфейса. Здесь храним сценарии, отчеты и артефакты (скриншоты/загрузки).

### Структура
- `cases/` — сценарии автотестов (TS/JS md‑скрипты или шаги для MCP)
- `reports/` — отчеты прогонов, логи
- `.playwright-mcp/` — скриншоты и загружаемые файлы (csv/xls и т. п.)
- `аудит1/` — результаты первого аудита (см. `аудит1/аудит.md`)

### Правила хранения артефактов
- Все скриншоты и скачанные файлы складываем в `auto-test/.playwright-mcp/`.
- Имена файлов понятные и стабильные: `orders.png`, `orders-export.png`, `integrations-settings.png` и т. п.
- Для повторных прогонов допускаются суффиксы с датой/временем или инкрементом.

### Запуск автотестов
Автотесты запускаются локально через сценарии Playwright MCP из этой папки.
- Перед запуском убедитесь, что фронтенд доступен на `http://localhost:3000`.
- Примеры команд (PowerShell):

```powershell
# пример: прогон кейсов (заглушка — дополним, когда добавим реальные скрипты)
# ./cases/run-all.ps1
```

В будущем добавим готовые скрипты (`run-all.ps1`, `smoke.ps1`) и экспорт отчётов в `reports/`.

### Краткая сводка задач (MVP)
- Фронтенд:
  - Таблицы: группировка/агрегации, сохранение состояния (сделано частично), экспорт CSV/XLS, мобильные карточки
  - Диалоги: a11y (`aria-labelledby/aria-describedby`), фокус-трап, закрытие по Esc
  - Навигация: крошки, тултипы, глобальный поиск (стабильные селекторы)
  - Микро‑анимации и полиш интерфейса
- Бэкенд:
  - Эндпоинты API v1 (extract/hardware/cam/validate/approve/1c/export)
  - Очереди CAM задач (Redis), стейты (Created→Processing→Completed/Failed)
  - Хранилище артефактов (Object Storage) с presigned URL (TTL ≤ 15 мин)
  - RAG по фурнитуре (эмбеддинги → YDB Vector → ранжирование)
